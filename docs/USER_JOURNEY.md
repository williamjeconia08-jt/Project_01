# SmartCent User Journey

> Version: 1.0
> Status: Active
> Last Updated: July 2026

---

# Purpose

This document describes the complete journey a teenager experiences while using SmartCent.

Rather than focusing on technical implementation, it explains how users interact with the platform from their first launch through long-term engagement.

Every feature, screen, database entity, and API endpoint should support this journey.

---

# User Persona

## Primary User

**Name:** Alex (Example)

**Age:** 14

**Education:** Secondary School Student

**Experience with Finance:** Beginner

**Device:** Android Smartphone

### Goals

- Learn how money works
- Understand budgeting
- Save virtual money
- Earn rewards
- Improve financial confidence

### Frustrations

- Traditional lessons are boring.
- Financial concepts seem confusing.
- Doesn't know where to begin.
- Wants learning to feel interactive.

---

# The SmartCent Journey

## Stage 1 – Discovery

The user hears about SmartCent through:

- School
- Friends
- Parents
- Social media
- Teachers

### Goal

Create curiosity.

---

## Stage 2 – Installation

The user downloads SmartCent.

### First Impression

The application should immediately communicate:

- This is fun.
- This is educational.
- This is easy to use.
- I want to continue.

---

## Stage 3 – Welcome Experience

The user opens SmartCent.

The app introduces:

- What SmartCent is
- Why financial literacy matters
- How learning works
- What rewards users can earn

### Goal

Build excitement.

---

## Stage 4 – Account Creation

The user creates an account.

Information collected:

- Username
- Email
- Password

Optional:

- Avatar

The process should be quick and simple.

---

## Stage 5 – Profile Setup

The user chooses:

- Avatar
- Display name

A short tutorial explains:

- XP
- Levels
- Wallet
- Badges
- Dashboard

---

## Stage 6 – First Lesson

The first lesson introduces:

"What is Money?"

The lesson is:

- Short
- Interactive
- Beginner friendly

---

## Stage 7 – First Quiz

The user completes a quiz based on the lesson.

After submission the user immediately sees:

- Score
- Correct answers
- XP earned

Positive reinforcement is essential.

---

## Stage 8 – First Reward

The user earns:

- First XP
- First Badge
- First Achievement

The app celebrates the accomplishment.

The goal is to create motivation.

---

## Stage 9 – Dashboard

The dashboard becomes the user's home.

It displays:

- Current level
- XP
- Wallet balance
- Learning streak
- Next lesson
- Active goals

The dashboard should clearly answer:

"What should I do next?"

---

## Stage 10 – Daily Learning

Users continue by:

- Completing lessons
- Taking quizzes
- Earning XP
- Unlocking levels
- Collecting badges

Learning should become a habit.

---

## Stage 11 – Financial Practice

Users begin practical activities.

Examples:

Create a budget.

Save SmartCoins.

Track spending.

Complete financial challenges.

These activities reinforce previous lessons.

---

## Stage 12 – Progress Tracking

Users can always view:

- Completed lessons
- Quiz history
- Badges
- Wallet history
- Budget performance
- Learning streak

Progress should feel visible and meaningful.

---

## Stage 13 – Level Progression

As users earn XP they unlock:

- New lessons
- New challenges
- New achievements

Progression should encourage continued learning rather than competition.

---

## Stage 14 – Long-Term Engagement

Users continue improving through:

- Daily challenges
- Advanced lessons
- New financial topics
- Special events (future)
- Personalized recommendations (future)

The objective is lifelong learning rather than short-term entertainment.

# Emotional Journey

Throughout the experience, SmartCent should guide users through these emotions:

Curiosity

↓

Confidence

↓

Achievement

↓

Motivation

↓

Growth

↓

Mastery

---

# Design Principles

Every screen should answer:

- Where am I?
- What can I do?
- Why does it matter?
- What should I do next?

Navigation should always feel predictable.

---

# Learning Loop

The SmartCent learning cycle is:

Learn

↓

Practice

↓

Challenge

↓

Reward

↓

Reflect

↓

Improve

↓

Repeat

This loop forms the foundation of the entire platform.

---

# Journey Success Criteria

A successful user journey means that users:

- Understand financial concepts.
- Feel motivated to continue learning.
- Build confidence over time.
- Develop healthy financial habits.
- Enjoy returning to SmartCent regularly.

---

# Conclusion

The user journey defines the experience SmartCent aims to deliver.

Every feature, database entity, API endpoint, and interface should exist to support this journey.

If a proposed feature does not improve or reinforce this journey, it should be reconsidered before implementation.
